# SARIF Workbench

**Опенсорсный инструмент для работы с результатами SAST-анализа.**

Просматривайте находки CodeQL, Semgrep, Svace и любых других SARIF-совместимых анализаторов.
Выполняйте AI-триаж с локальными или облачными LLM.
Экспортируйте PDF-отчёты. Работает полностью офлайн — создан для air-gap и корпоративных контуров.

[Быстрый старт](guide/quickstart.md){ .md-button .md-button--primary }
[GitHub](https://github.com/DumDereDum/sarif-workbench){ .md-button }

## Зачем SARIF Workbench?

Современные статические анализаторы генерируют сотни находок за один прогон. Команды тратят дни на ручной просмотр ложных срабатываний. SARIF Workbench решает это:

| Проблема | Решение |
|---|---|
| Слишком много находок для ручного просмотра | AI-триаж классифицирует каждую находку как TP / FP / Uncertain |
| Отчёты в Excel вместо стандартного формата | Экспорт PDF одним кликом в формате, совместимом со Svacer |
| Разные анализаторы — разные форматы | Единая загрузка SARIF 2.1.0 с нормализацией severity |

## Ключевые возможности

<div class="grid cards" markdown>

-   :material-magnify: **Единый браузер находок**

    ---

    Фильтрация по severity, вердикту, правилу, файлу или полнотекстовый поиск. Нажмите на находку — увидите сниппет кода, описание CWE, git blame и историю триажа.

-   :material-robot: **AI-триаж**

    ---

    Вердикты AI стримятся через SSE на базе DeepSeek. Выберите стратегию промпта для каждого прогона: сбалансированный, с уклоном в FP или в TP.

-   :material-file-pdf-box: **PDF-отчёты**

    ---

    Серверная генерация PDF через WeasyPrint. Макет соответствует стилю Svacer — сниппеты кода, вердикт-бейджи, обоснование триажа и оглавление по правилам.

-   :material-lock: **Air-gap без компромиссов**

    ---

    Сервер, Web UI и CLI не имеют внешних сетевых зависимостей. Развёртывание одной командой `docker compose up`.

-   :material-console: **CLI для CI/CD**

    ---

    `swb-cli` запускается на CI-раннере, где доступен исходный код. Обогащает SARIF сниппетами, git-метаданными и отпечатками — затем загружает пакет на сервер.

</div>

## Как это работает

```
┌─────────────┐  enrich  ┌───────────────────┐  upload  ┌─────────────┐
│  SARIF-файл │ ────────► │  .swbmeta.json    │ ────────► │   Сервер    │
│  от SAST    │          │  (сайдкар)        │          │  + Web UI   │
└─────────────┘          └───────────────────┘          └─────────────┘
```

1. **`swb-cli enrich`** читает SARIF на CI-раннере, извлекает git-метаданные, сниппеты кода и стабильные отпечатки для сопоставления прогонов. Создаёт `report.sarif.swbmeta.json` рядом с исходным файлом. Исходный SARIF не изменяется.

2. **`swb-cli upload`** отправляет пару SARIF + сайдкар на сервер. Сервер автоматически определяет имя проекта из git-provenance.

3. **Web UI** — находки сразу доступны в браузере. Фильтруйте, выносите вердикты, запускайте AI-анализ и скачивайте PDF-отчёт.

## Поддерживаемые анализаторы

Любой инструмент, который выводит [SARIF 2.1.0](https://docs.oasis-open.org/sarif/sarif/v2.1.0/sarif-v2.1.0.html):

**Проверено:** CodeQL · Semgrep · Svace · Checkmarx · Fortify (через экспорт в SARIF)

## Быстрый старт

```bash
# 1. Запустить сервер
git clone https://github.com/DumDereDum/sarif-workbench && cd sarif-workbench
docker compose up

# 2. Обогатить и загрузить SARIF (запускать там, где есть исходный код)
swb-cli enrich report.sarif --repo-root .
swb-cli upload report.sarif --server http://localhost:8000

# 3. Открыть интерфейс
open http://localhost:5173
```
